# Phase 2: Mechanics

**Outcome:** "I can verify Python and build a working agent"

**Estimated time:** ~10–11 hours
**Dependencies:** Phase 1 complete.

## Lessons

1. **[Lesson-01-Flow-Models-and-Verification.md](Lesson-01-Flow-Models-and-Verification.md)** — Control flow, dataclasses, and pytest — turning assertions into an automated verification layer.
2. **[Lesson-02-Errors-Validation-and-Independence.md](Lesson-02-Errors-Validation-and-Independence.md)** — Iterating on AI output, error handling, Pydantic validation, and debugging well enough to drive TDG solo.
3. **[Lesson-03-The-Python-Object-Model.md](Lesson-03-The-Python-Object-Model.md)** — Classes, inheritance vs. composition, special methods, and decorators/protocols.
4. **[Lesson-04-Building-Your-First-OpenAI-Agent.md](Lesson-04-Building-Your-First-OpenAI-Agent.md)** — The SDK's three primitives (Agent, Runner, function_tool), typed context, and combining agents.
5. **[Lesson-05-Multi-Agent-Reliability.md](Lesson-05-Multi-Agent-Reliability.md)** — Handoffs, guardrails, sessions/memory, and tracing — what separates a demo from something reliable.
6. **[Lesson-06-Grounding-Your-Agent-MCP-RAG-Capstone.md](Lesson-06-Grounding-Your-Agent-MCP-RAG-Capstone.md)** — Connecting the agent to real tools via MCP and real knowledge via RAG; the Digital FTE capstone.
7. **[Lesson-07-Comparing-Agent-Frameworks.md](Lesson-07-Comparing-Agent-Frameworks.md)** — The same architecture expressed in Google ADK, the Claude API agentic loop, and the Claude Agent SDK.
8. **[Lesson-08-MCP-Architecture-Transports-Tools-Resources.md](Lesson-08-MCP-Architecture-Transports-Tools-Resources.md)** — MCP's Host-Client-Server model, transports, and the Tools/Resources primitives.
9. **[Lesson-09-Using-MCP-Prompts-Config-Debugging.md](Lesson-09-Using-MCP-Prompts-Config-Debugging.md)** — The Prompts primitive, wiring a server into a client, community servers, and debugging.
